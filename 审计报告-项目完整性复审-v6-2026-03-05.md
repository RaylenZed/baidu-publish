## 4. 最小修复建议

1. 先修 P0：
   - `useTaskLogStream` 补齐 `task_id` 字段映射（或放宽 `TaskLog` 类型）
   - 清理 `PoolsView` 未使用声明，恢复 `vue-tsc` 通过

2. 修调度边界：
   - 对 `next_fire_at < threshold` 的 schedule 执行“跳过补执行但推进 next_fire_at 到未来”策略，避免永久饿死

3. 收敛宪法一致性：
   - `PoolsView` 改为从后端品类 API 获取，不在前端硬编码品类

4. 跑通测试基线：
   - 安装依赖后至少保证 `pytest -q` 可执行
   - 补 3 条关键集成测试：任务幂等并发、WS 日志流、misfire 恢复边界
