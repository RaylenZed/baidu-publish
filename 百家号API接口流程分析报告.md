# 百家号 AIGC 自动化软文发布全流程接口分析报告

这份接口日志完整地勾勒出了一套**“从AI生成到图文处理，最终在百家号正式发布”**的自动化软文发布流转全过程。根据提供的《百家号相关接口.txt》日志文件梳理，整个接口的流转序列可以清晰地分为四个核心阶段：

---

### 第一阶段：AI 创作预发布阶段（灵感与草稿）

这一阶段的主要目的是唤醒百度的大模型，并发送提示词（Prompt）获取文章的基础正文。

1. **生成对话 (`/aigc/saas/pc/v1/aiNews/createDialogue`)**
   - **请求方式**：POST
   - **作用**：初始化当前的 AI 写作对话，相当于新建一个写作聊天窗口，获取 `dialogue_id` 为接下来的 Prompt 下发做准备。

2. **生产文章 (`/aigc/saas/pc/v1/aiNews/chat`)**
   - **请求方式**：POST
   - **作用**：基于创建的 `dialogue_id` 发送用户的具体需求指令（例如 query: "我是一个百家号博主，帮我生成一篇关于生鲜食品的文章！"）。
   - **机制**：接口以**流式输出（SSE）**的方式，不断返回片段数据，最终拼装成由 Markdown 语法构成的 AI 生成文章正文 (`content`)、标题 (`title`) 以及 `active_article_id`。

---

### 第二阶段：草稿转换与落库阶段

这一阶段的目的是将上一步在对话框里生成的虚拟内容，正式落实在百家号创作者后台的草稿箱中。

3. **获取文章详情 (`/aigc/saas/pc/v1/aiNews/articleDetail`)**
   - **请求方式**：GET
   - **作用**：凭借上一步生成的 AI 对话记录，查询这篇生成文章的完整具体内容，确认内容无误。

4. **生成正式文章ID (`/aigc/saas/pc/v1/aiNews/publishAiNews`)**
   - **请求方式**：POST
   - **作用**：这是关键的**“身份转换”步骤**。将刚才纯粹的 AI 对话内容草稿转换进入百家号的正规草稿箱体系。成功后后端会返回一个正式的、可用于后续上传图片和最终提交的 `article_id`（如 `1857838114108046682`）。

---

### 第三阶段：文章配图与封面处理阶段

在这个阶段，系统为纯文本软文丰富图文表现，方案中采取了两种并行的方式寻找封面和配图：

#### 分支 A：调用正版图库（常规检索流程）
5. **搜索正版图片 (`/aigc/bjh/pc/v1/picSearch`)**
   - **请求方式**：POST
   - **作用**：根据提取的文章关键词（如“汽车”或“蔬菜”），在百家号的版权图库中检索对应的图片列表，并返回多规格的图片URL (`big_thumb_url`, `bjh_watermark_url` 等)。
6. **选择封面自动裁剪 (`/materialui/picture/auto_cutting`)**
   - **请求方式**：POST
   - **作用**：确定某张版权原图后，调用该接口将其自动裁剪为符合百家号封面规定比例的图片，准备用于最终发布配置。

#### 分支 B：调用 AI 文生图引擎（AIGC绘图流程）
7. **AI生成图片 (`/writebrain/text2img/generate_img`)**
   - **请求方式**：POST
   - **作用**：直接给 AI 传入一段具体的场景描述语句作为 Prompt（例如寿光田间摘蔬菜的画面），让 AI 自行绘制并返回多张候选的 `img_url`。
8. **选择并转码 Ai 封面 (`/pcui/picture/processproxy`)**
   - **请求方式**：POST
   - **作用**：将选定的 AI 候选图片进行 Base64 或代理转码处理，将其引入至百家号的素材库中。
   - *(注：日志中还包含一个 `action=copy` 接口用于固化所选图片的最终 CDN 存储地址)*

---

### 第四阶段：正式组装与提交流程阶段

最后也是最关键的步骤，收集文本、摘要、设置分类、嵌入封面图，打包后发往终审。

9. **获取内容分类库 (`/pcui/article/cateusercms`)**
   - **请求方式**：GET
   - **作用**：在正式发布前，获取并验证百家号后端的文章分类字典（例如确认文章将要落地的 “军事” 或 “农业” 频道标识）。

10. **正式发文发布 (`/pcui/article/publish`)**
    - **请求方式**：POST
    - **作用**：**发文总控收口**。经过前面所有的准备，组装最终的表单数据包。
    - **涵盖核心参数**：
      - `title`, `content`（经过 HTML 标签重排的正文）
      - `abstract`（提炼的摘要片段）
      - `cate_user_cms`（文章类目）
      - `cover_images`（第三阶段处理并格式化好的封面图 JSON 列表）
      - 以及其他发文配置属性（如是否开启免费朗读体验 `ai_tts=1` 等）。

---

### 总结点评

这份接口文件清晰展示了一条全自动化闭环链路：
**唤醒AI -> 注入Prompt撰写 -> 注册为正式草稿 -> 系统级搜图/AI生图并裁剪封面 -> 组装成富文本与多媒体混合表单 -> 投递给百家号最终落库。**

这条链路上百家号对 API 进行了解耦，使得前期的 AI 内容探索和后期的标准图文组装可以分批进行，这为外部（如通过 Python 或 Dify 工作流）开发多账号并发矩阵控制提供了极高可行性。
