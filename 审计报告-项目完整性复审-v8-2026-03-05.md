## 3. 主要问题（按严重级）

### P1

1. 密码哈希链路依赖组合仍不稳定（交付阻塞）
   - `requirements.txt` 仅固定 `passlib[bcrypt]==1.7.4`，未限制 `bcrypt` 版本：
     - `src/backend/requirements.txt:24`
   - 实际安装到 `bcrypt 5.0.0` 后，`hash_password()` 路径在测试中失败：
     - `src/backend/app/core/security.py:25-33`
   - 影响：认证/登录相关能力在新环境下不具备可验证稳定性，CI 难以稳定全绿。

2. 并发控制与配置不一致（架构约束未闭环）
   - Worker 仍固定 `--concurrency=3`：
     - `docker-compose.yml:83-86`
   - 但系统设置有 `max_concurrent_accounts`，默认/期望按配置控制：
     - `src/backend/app/models/system_settings.py:82-88`
   - 影响：运行态并发与管理台配置不一致，违背“同账号串行 + 跨账号受控并发”的运维预期。

### P2

1. 设置页未覆盖后端已支持的全部关键字段
   - 前端已展示：`aigc_model / max_concurrent_accounts / daily_limit / task_timeout_minutes / generate_timeout / polish_timeout / wecom_webhook / notify_level`
   - 前端未展示：`run_mode / account_delay / cover_timeout / publish_timeout / draft_timeout`
   - 证据：
     - 后端字段：`src/backend/app/schemas/settings.py:24-35`
     - 前端表单：`src/frontend/src/views/SettingsView.vue:46-70`

2. 变量池页面仍缺少显式“新增/删除条目”交互入口
   - 当前仅可编辑既有 `items` 字段并保存：
     - `src/frontend/src/views/PoolsView.vue:167-187`
