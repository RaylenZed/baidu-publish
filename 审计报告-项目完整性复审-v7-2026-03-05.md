## 3. 主要问题（按严重级）

### P1

1. WebSocket 实时日志会丢增量日志（同 ID 去重导致）
   - 后端 Worker 推送日志时 `log_id` 默认值恒为 `0`
     - `src/backend/app/workers/tasks.py:579-585`
     - `src/backend/app/ws/task_log_stream.py:146,155`
   - 前端按 `id` 去重，后续增量日志（同为 `id=0`）被过滤
     - `src/frontend/src/composables/useTaskLogStream.ts:78-80`
   - 影响：任务运行中的实时日志显示不完整。

2. 依赖兼容性导致测试基线不稳定（密码哈希链路）
   - 现状：`requirements.txt` 仅声明 `passlib[bcrypt]>=1.7.4`，未约束 `bcrypt` 上限
     - `src/backend/requirements.txt:24`
   - 隔离环境安装后命中 `passlib 1.7.4 + bcrypt 5.0.0` 组合，`hash_password()` 相关测试失败
   - 失败入口：`src/backend/app/core/security.py:29`、`src/backend/tests/unit/test_security.py:22,28`、`src/backend/tests/integration/test_auth.py:61`
   - 影响：在新环境中可能出现登录/鉴权相关不稳定行为，且 CI 难以稳定绿。

3. 并发控制仍与系统配置值不一致
   - `max_concurrent_accounts` 可配置，但 worker 并发固定为 3
   - 证据：`docker-compose.yml:85`
   - 影响：配置与实际执行偏差，难以满足“同账号串行 + 跨账号受控并发”的运维预期。

### P2

1. 前端模块完整性仍有缺口
   - 账号导入/导出/批量检测未落地 UI
   - 定时任务编辑（update）未落地 UI
   - 系统设置页未覆盖全部后端字段（如 `run_mode`、`account_delay`、`cover/publish/draft_timeout`）

2. 变量池页面目前仅支持编辑既有条目，未见新增/删除条目的操作入口
   - 证据：`src/frontend/src/views/PoolsView.vue:167-187`
