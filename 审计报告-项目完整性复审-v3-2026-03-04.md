## 1. 发现（按严重级）

### P0（仍阻塞“可按文档一键跑通”）

1. **快速启动链路与实际部署文件不一致，默认会卡在环境配置**
   - `docker-compose.yml` 指定 `env_file: ./src/backend/.env`（`docker-compose.yml:53`, `:76`, `:99`）
   - 但仓库提供的是根目录 `.env.example`，README 指导 `cp .env.example .env`（`README.md:35`）
   - 当前仓库无 `src/backend/.env`，且 `.env.example` 未覆盖必填项（如 `JWT_SECRET_KEY`）
   - 结论：按 README 走，无法保证 compose 直接可启动

2. **`.env.example` 与后端运行要求不匹配**
   - `Settings` 要求 `JWT_SECRET_KEY` 必填（`src/backend/app/core/config.py:32`）
   - `.env.example` 未提供该变量（`./.env.example:1-46`）
   - `.env.example` 的 `DATABASE_URL` 为 `postgresql://...`（`./.env.example:14`），而后端注释与实现是 async 引擎链路（`src/backend/app/core/config.py:41`）
   - 结论：模板配置不能直接满足后端启动条件

### P1（核心质量风险）

1. **`CORS_ORIGINS`“逗号串”兼容修复未生效**
   - 代码新增了解析 validator（`src/backend/app/core/config.py:51-63`）
   - 但复验时 `CORS_ORIGINS=http://localhost:5173,http://localhost:80` 仍抛 `SettingsError`
   - 原因是 `pydantic-settings` 对 `list[str]` 先做复杂类型解码，未到 validator 就失败
   - 结论：该问题仍未彻底关闭

2. **手动发布失败状态仍可能被回滚，状态与审计一致性有风险**
   - `manual_publish` 失败时设 `publish_failed` 后立即抛异常（`src/backend/app/services/article_service.py:223-230`）
   - 路由未做失败分支提交（`src/backend/app/api/routes/articles.py:105-138`）
   - `get_db` 在异常时会回滚事务（`src/backend/app/core/database.py`）
   - 结论：失败态可能无法稳定落库

3. **账号导入循环内 `rollback()` 仍会影响前序成功项统计一致性**
   - `import_accounts` 单条异常即 `await db.rollback()`（`src/backend/app/services/account_service.py:353-356`）
   - 结论：可能出现“计数显示成功，但事务已回滚”的不一致

4. **调度 Misfire 策略与 PRD 24h 约束仍不严格一致**
   - 常规触发 `fire_enabled_schedules` 对所有 `next_fire_at <= now` 生效（`src/backend/app/services/schedule_service.py:255-278`）
   - 未限定 24h 窗口；而 `recover_misfire` 的筛选逻辑又依赖 `last_fired_at`（`:280-306`）
   - 结论：与“仅补执行 24h 内错过触发一次”仍有偏差风险

### P2（体验/契约一致性问题）

1. 任务列表前端显示 `account_name`，但任务响应无该字段，账号列会空白。
   - 前端：`src/frontend/src/views/TasksView.vue:32`
   - 后端 `TaskResponse` 无 `account_name`：`src/backend/app/schemas/task.py`

2. `viewLogs/viewArticle` 仍使用 `window.open('/tasks/{id}/logs')` 这类前端不存在页面路由。
   - 示例：`src/frontend/src/views/TasksView.vue:108-110`

3. 账号页仍保留“接口失败时硬编码品类”降级逻辑，不满足“前端不硬编码枚举”的宪法约束。
   - `src/frontend/src/views/AccountsView.vue:55-61`

4. README 仍有多处陈旧信息：
   - WebSocket 路径写成 `/api/v1/ws/...`（`README.md:157`），实际是 `/ws/tasks/{id}/logs`
   - 备份命令用户 `bjh`（`README.md:219`）与 compose 的 `bjh_user` 不一致（`docker-compose.yml:18`）
